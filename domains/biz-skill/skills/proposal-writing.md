# 사업계획서 작성

정부지원사업 심사기준에 최적화된 사업계획서를 작성하여 서면평가 및 발표평가에서 높은 점수를 획득한다.

---

## 규칙

1. **심사항목별 배점에 맞춰 분량 배분**: 배점 30점 항목에는 전체의 30% 분량을 할당한다
2. **한 페이지 한 메시지**: 각 페이지는 하나의 핵심 메시지만 전달한다
3. **도표 적극 활용**: 텍스트 나열보다 표, 그래프, 다이어그램으로 시각화한다
4. **수치 근거 필수**: 모든 주장에는 출처 있는 정량적 데이터를 첨부한다
5. **심사위원 관점**: 10분 내에 핵심을 파악할 수 있도록 구조화한다

---

## 공통 심사항목별 작성 가이드

### 1. 문제 인식 및 해결 방안 (혁신성) - 배점 20~30

```
작성 구조:
[1] 현재 시장/고객의 핵심 문제 (Pain Point) - 구체적 사례
[2] 기존 해결 방식의 한계 - 왜 아직 해결되지 않았는가
[3] 우리의 해결 방안 - 기술/서비스의 차별적 접근
[4] 혁신성 근거 - 특허, 논문, 기술적 우위, 새로운 방법론
```

**팁**: "~최초", "~유일" 표현은 근거 없이 사용하면 감점 요인이 된다. 구체적 차별점을 기술한다.

### 2. 시장 분석 및 사업화 전략 (시장성) - 배점 20~30

```
작성 구조:
[1] TAM/SAM/SOM 시장규모 (출처 포함)
[2] 시장 성장률 및 트렌드
[3] 목표 고객군 정의 및 세분화
[4] Go-to-Market 전략 (초기 고객 확보 → 확장)
[5] 매출 계획 (3개년) - 보수적/기본/낙관 시나리오
```

**팁**: 매출 계획은 반드시 고객수 x 객단가 x 전환율로 역산 가능한 구조로 제시한다.

### 3. 기술 개발 계획 (기술성) - 배점 10~30

```
작성 구조:
[1] 핵심 기술 개요 및 기술 수준
[2] 기술 개발 로드맵 (마일스톤별)
[3] 기술적 차별점 및 진입장벽
[4] 지식재산권 확보 계획 (특허, 영업비밀)
[5] 기술 검증 현황 (프로토타입, PoC, MVP)
```

**팁**: R&D 사업일수록 기술성 배점이 높다. 기술 TRL(기술성숙도) 수준을 명시하면 유리하다.

### 4. 실현 가능성 (팀역량, 일정, 자원) - 배점 15~25

```
작성 구조:
[1] 대표자 역량 - 관련 경력, 학위, 수상, 특허
[2] 팀 구성 - 핵심 인력 역할과 역량
[3] 추진 일정 - 월별/분기별 마일스톤 (간트차트)
[4] 인프라/자원 - 사무실, 장비, 협력기관
[5] 리스크 관리 - 예상 위험과 대응 방안
```

**팁**: 대표자의 해당 분야 경험이 가장 중요하다. 관련 없는 화려한 이력보다 직접적 경험을 강조한다.

### 5. 기대효과 (매출, 고용, 사회적 가치) - 배점 10~15

```
작성 구조:
[1] 정량적 성과 목표 - 매출, 고용, 수출
[2] 기술적 성과 - 특허, 인증, 논문
[3] 사회적 가치 - 일자리 창출, 지역경제, ESG
[4] 후속 성장 계획 - 투자유치, 해외진출, 후속 R&D
```

## 사업계획서 목차 템플릿 (20~30페이지)

```
1. 요약서 (Executive Summary)                    1p
2. 기업 개요                                      1~2p
   - 기업 현황, 대표자 소개, 조직도
3. 문제 정의 및 해결 방안                          3~5p
   - 시장 문제, 기존 한계, 우리의 솔루션, 혁신성
4. 제품/서비스 소개                                3~4p
   - 핵심 기능, 기술 구조, 차별점, 프로토타입
5. 시장 분석                                      3~4p
   - TAM/SAM/SOM, 경쟁사 분석, 포지셔닝
6. 사업화 전략                                    3~4p
   - BM, 가격정책, 마케팅, 고객확보 전략
7. 기술 개발 계획                                  2~3p
   - 기술 로드맵, IP 전략, TRL
8. 추진 일정 및 팀                                2~3p
   - 간트차트, 인력계획, 협력기관
9. 사업비 편성                                    2~3p
   - 비목별 편성표, 산출근거
10. 기대효과 및 성장 계획                          1~2p
    - 정량 목표, 사회적 가치
```

## 작성 팁

| 항목 | DO | DON'T |
|------|-----|-------|
| 문제 정의 | 고객 인터뷰/데이터 기반 | 추상적 사회 문제 나열 |
| 시장규모 | 출처 있는 TAM/SAM/SOM | 근거 없는 "X조원 시장" |
| 차별점 | 기능/성능 정량 비교표 | "세계 최초", "유일한" |
| 팀역량 | 관련 분야 구체적 경험 | 무관한 화려한 스펙 |
| 매출계획 | 고객수 x 단가 역산 | 시장점유율 X% 가정 |
| 기대효과 | 보수적 수치 + 근거 | 비현실적 낙관 전망 |

### 킬러 슬라이드 3가지

1. **문제-해결 한 장**: 고객 고통 → 기존 한계 → 우리 솔루션 (비포/애프터)
2. **경쟁우위 비교표**: 핵심 기능 5가지 O/X 비교 (자사만 O인 항목 필수)
3. **성장 로드맵**: 3개년 매출/고객/고용 성장 그래프

---

## 요약서(Executive Summary) 1장 작성법

```
구성 (A4 1페이지):
[1] 한 줄 소개 - 무엇을 하는 회사인가 (1문장)
[2] 문제 - 어떤 문제를 해결하는가 (2~3문장)
[3] 솔루션 - 어떻게 해결하는가 (2~3문장)
[4] 시장 - 시장규모와 성장률 (TAM/SOM, 1~2문장)
[5] 차별점 - 왜 우리인가 (핵심 1~2가지)
[6] 팀 - 왜 이 팀이 할 수 있는가 (1~2문장)
[7] 목표 - 이 사업으로 달성할 것 (정량 목표 3가지)
```

**원칙**: 요약서만 읽고도 사업 전체를 이해할 수 있어야 한다. 심사위원은 요약서로 첫인상을 결정한다.

## 체크리스트

- [ ] 심사항목별 배점을 확인하고 분량을 배분했는가
- [ ] 모든 주장에 출처 있는 정량 데이터를 첨부했는가
- [ ] 경쟁사 비교표에서 명확한 차별점이 드러나는가
- [ ] 매출 계획이 고객수 x 단가로 역산 가능한가
- [ ] 추진일정이 마일스톤별로 구체적인가 (간트차트)
- [ ] 대표자/팀의 관련 분야 역량이 명확히 기술되었는가
- [ ] 요약서 1장으로 사업 전체를 파악할 수 있는가
- [ ] 페이지마다 하나의 핵심 메시지만 전달하는가
- [ ] 제3자가 읽어도 이해 가능한 문장인가 (전문용어 설명)
- [ ] 오탈자, 수치 불일치, 서식 오류를 최종 점검했는가
