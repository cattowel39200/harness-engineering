# MVP/프로토타입

최소 기능 제품(MVP)의 범위를 정의하고 개발 계획과 데모 시나리오를 수립하여 핵심 가설을 검증한다.

---

## 규칙

1. **가설 검증 목적**: MVP는 완성품이 아니라 핵심 가설을 검증하기 위한 최소 기능 제품이다
2. **범위 절제**: Must-have 기능만 포함하고 Nice-to-have는 과감히 제외한다
3. **시연 가능성 확보**: 심사위원 앞에서 실제 동작을 보여줄 수 있어야 한다
4. **비용 효율**: 정부사업 시제품 예산 내에서 실현 가능한 수준으로 계획한다
5. **피드백 순환**: 개발 → 테스트 → 피드백 → 개선의 반복 주기를 설계한다

---

## MVP 정의 및 핵심 가설

```
MVP = "우리 솔루션이 고객 문제를 해결하는가"를 검증하는 최소 단위
가설: [1] 문제가 실재하는가 [2] 솔루션이 해결하는가 [3] 지불 의향이 있는가
```

### MVP 유형 선택

| 유형 | 설명 | 적합한 경우 | 기간/비용 |
|------|------|------------|----------|
| 랜딩페이지 | 서비스 소개 + 사전등록 | 수요 검증 | 1~2주 / 50만원 |
| 목업/클릭 프로토타입 | Figma로 화면 흐름 시연 | 초기 발표용 | 2~3주 / 100만원 |
| 노코드 MVP | 버블, 아임웹 등으로 구현 | 빠른 시장 테스트 | 3~4주 / 200만원 |
| 기능 제한 MVP | 핵심 기능만 직접 개발 | 기술 검증 필요 | 2~3개월 / 500만원+ |
| 하드웨어 시제품 | 3D프린팅, PCB 제작 | 제조업 | 3~6개월 / 1000만원+ |

---

## MVP 범위 설정

```
Must-have (MVP 포함): 핵심 기능 1~3가지, 기본 사용자 플로우, 데모 화면
Should-have (이후): 관리자 대시보드, 결제 시스템, 알림
Could-have: 소셜 로그인, 다국어, 고급 분석
Won't-have: 모바일 앱(웹 우선), AI 고도화, 대규모 트래픽
```

---

## 개발 방식 선택

### 직접 개발 vs 외주 vs 노코드

| 항목 | 직접 개발 | 외주 개발 | 노코드/로우코드 |
|------|-----------|-----------|----------------|
| 비용 | 인건비(정부사업) | 위탁개발비 | 구독료 + 인건비 |
| 기간 | 2~4개월 | 2~3개월 | 2~4주 |
| 품질 | 높음 (통제 가능) | 편차 큼 | 기능 제약 |
| IP | 자사 소유 | 계약 확인 필수 | 플랫폼 종속 |
| 유지보수 | 자체 | 추가 비용 | 플랫폼 의존 |
| 적합 | 기술 기반 창업 | 비기술 창업자 | 수요 검증 단계 |

**정부사업 주의사항**: 외주 개발 시 위탁개발비로 편성. 계약서에 IP 귀속, 소스코드 인도, 하자보증 기간 명시 필수.

## 시제품 개발 일정

```
Week 1~2:  기획 확정
           - 기능 정의서 작성
           - 화면 설계 (와이어프레임)
           - DB/API 설계

Week 3~4:  디자인 (UI 디자인, 디자인 시스템, 플로우 확정)
Week 5~8:  개발 (백엔드 API, 프론트엔드, 핵심 기능 통합)
Week 9~10: 테스트 (QA, 베타 사용자 5~10명, 버그 수정)
Week 11~12: 시연 준비 (데모 시나리오, 환경 세팅, 리허설)
```

## 데모 시나리오 작성 (심사위원 대상)

### 시나리오 구조 (3~5분)

```
[도입 30초] 고객 페르소나 소개 + 현재 겪는 문제
  "김 대리는 매일 2시간을 ___에 소비합니다"

[핵심 기능 2분] 솔루션이 문제를 해결하는 과정
  실제 화면에서 주요 기능 3가지를 순서대로 시연
  각 기능마다 "이전에는 ~, 이제는 ~" 비교

[결과 30초] 사용 후 효과/개선 수치
  "처리 시간이 2시간에서 10분으로 단축됩니다"

[마무리 30초] 향후 확장 계획 한 줄
```

### 데모 체크리스트

- 인터넷 없이도 동작하는 오프라인 백업 준비
- 더미 데이터로 현실감 있는 시연 데이터 세팅
- 에러 발생 가능 경로 회피하는 시나리오 설계
- 화면 해상도, 프로젝터 호환성 사전 확인
- 예상 소요 시간별 숏/미디엄/롱 버전 준비

## 사업비 연계

| 항목 | 비목 | 예상 비용 |
|------|------|-----------|
| UI/UX 디자인 | 외주용역비 | 200~500만원 |
| 외주 개발 | 위탁개발비 | 500~2,000만원 |
| 서버/인프라 | 기자재 구입비 | 100~300만원 |
| 도메인/SSL | 특허출원비/기타 | 10~20만원 |
| 테스트 비용 | 시제품 제작비 | 100~300만원 |
| 자체 개발 인건비 | 인건비 | 월 300~500만원/인 |

---

## 체크리스트

- [ ] 검증할 핵심 가설 1~3가지를 명확히 정의했는가
- [ ] Must-have 기능이 3개 이하로 절제되었는가
- [ ] 개발 방식(직접/외주/노코드)을 결정하고 근거가 있는가
- [ ] 개발 일정이 정부사업 일정 내에 현실적으로 가능한가
- [ ] 데모 시나리오가 3~5분 이내로 핵심을 보여주는가
- [ ] 오프라인 백업 등 데모 실패 대비책이 있는가
- [ ] 시제품 관련 사업비가 적절한 비목에 편성되었는가
- [ ] MVP 이후 확장 계획(로드맵)이 연결되어 있는가
