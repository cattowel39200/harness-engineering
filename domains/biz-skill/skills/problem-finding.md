# 문제 발견

트렌드 탐색에서 도출한 기회 후보를 기반으로, 실제 고객이 겪는 구체적인 문제를 발견하고 명확하게 정의한다.

---

## 규칙

1. **실제 고객 관점**: 내가 만들고 싶은 것이 아니라 고객이 겪는 실제 불편/고통에서 출발한다
2. **기존 해결책의 한계 분석**: 문제가 이미 해결되었는지, 기존 솔루션의 부족한 점은 무엇인지 파악한다
3. **구체적 문제 정의**: "불편하다"가 아니라 "A 상황에서 B를 할 때 C 때문에 D만큼 손해가 발생한다" 수준으로 구체화한다
4. **규모 추정 필수**: 이 문제를 겪는 사람/기업의 수를 반드시 추정한다
5. **문제와 솔루션 분리**: 이 단계에서는 해결책을 생각하지 않는다. 문제 정의에만 집중한다

---

## 문제 발견 프레임워크

### 1. 5WHY 분석
표면적 문제에서 시작하여 "왜?"를 5번 반복하여 근본 원인에 도달한다.

```
문제: 중소 제조업체의 불량률이 높다
왜 1? 품질검사가 수작업이라 누락이 발생한다
왜 2? 자동검사 장비가 너무 비싸서 도입하지 못한다
왜 3? 기존 비전검사 장비가 대기업 맞춤형이다
왜 4? 중소기업용 저가 솔루션 시장이 형성되지 않았다
왜 5? AI 비전검사의 중소기업 적용 사례/모델이 부족하다
→ 근본 문제: 중소 제조업 맞춤형 AI 비전검사 솔루션 부재
```

### 2. 고객 여정 맵 (Customer Journey Map)
대상 고객의 업무/생활 흐름을 단계별로 나열하고, 각 단계에서의 불편/비효율을 찾는다.

```
[인지] → [탐색] → [비교] → [결정] → [구매] → [사용] → [유지보수] → [폐기/전환]
  각 단계: 행동 / 감정 / 페인포인트 / 기회
```

### 3. 페인포인트 매핑
발견한 문제들을 심각도(Impact)와 빈도(Frequency) 2축으로 매핑한다.

```
        높은 빈도
           |
   [관리필요]  |  [핵심 문제] ← 여기를 노린다
           |
  ─────────┼─────────
           |
   [무시가능]  |  [잠재 기회]
           |
        낮은 빈도
  낮은 심각도        높은 심각도
```

## 문제 유형 분류

### 효율성 문제 (느리거나 비싸다)
- 수작업으로 처리하여 시간이 오래 걸린다
- 비용이 과도하게 높다
- 중복 작업이 발생한다
- 데이터가 분산되어 통합 관리가 안 된다

### 접근성 문제 (어렵거나 못 쓴다)
- 기술/서비스가 존재하지만 특정 계층이 접근하지 못한다
- 전문지식이 필요하여 일반인이 사용할 수 없다
- 지역/환경적 제약으로 서비스가 불가하다
- 가격 장벽이 높다

### 품질 문제 (부족하거나 불만족)
- 기존 솔루션의 정확도/성능이 부족하다
- 사용자 경험(UX)이 불편하다
- 맞춤화/개인화가 되지 않는다
- 신뢰성/안정성이 떨어진다

### 규제/환경 변화 문제
- 새로운 규제로 기존 방식이 불가능해졌다
- 의무화된 사항을 이행할 수단이 부족하다
- 환경 변화(코로나, 기후변화 등)로 새로운 니즈가 발생했다

## 문제 진술서 템플릿

각 문제를 아래 형식으로 작성한다:

```
### 문제 #N: [문제 제목]

**대상 (누가)**:
- 주요 타겟: (예: 50인 미만 중소 제조업체)
- 추정 규모: (예: 국내 약 7만 개사)

**상황 (무엇을)**:
- 구체적 상황 설명
- 현재 대안/기존 해결책

**원인 (왜)**:
- 근본 원인 (5WHY 결과)
- 기존 솔루션이 해결하지 못하는 이유

**영향 (얼마나)**:
- 금전적 손실: (연간 OO만원/건)
- 시간 손실: (월 OO시간)
- 기타 영향: (안전, 품질, 만족도 등)

**문제 유형**: 효율성 / 접근성 / 품질 / 규제변화
**관련 트렌드**: (trend-scan에서 도출한 트렌드 참조)
**심각도**: 상/중/하
**빈도**: 상/중/하
```

## 문제 발견 정보 수집 방법

1. **데스크 리서치**: 산업 보고서, 뉴스 기사, 커뮤니티 불만 글
2. **현장 관찰**: 실제 업무 환경 방문, 작업 과정 관찰
3. **인터뷰**: 잠재 고객 3-5명 비구조화 인터뷰 (열린 질문)
4. **설문조사**: 문제 가설 검증을 위한 간단한 온라인 설문
5. **데이터 분석**: 공개 통계, 민원 데이터, 사고 통계 등

---

## 산출물

1. **문제 진술서 3-5개**: 위 템플릿에 따라 구체적으로 작성
2. **페인포인트 매핑**: 심각도 x 빈도 매트릭스에 문제 배치
3. **우선순위 문제 1-2개**: 핵심 문제 영역에 해당하는 문제 선정

---

## 체크리스트

- [ ] 트렌드 보고서의 기회 후보에서 출발했는가
- [ ] 고객 관점에서 문제를 서술했는가 (기술자 관점이 아닌)
- [ ] 5WHY 또는 고객 여정 맵을 활용하여 근본 원인을 파악했는가
- [ ] 기존 해결책이 있는지 조사하고 한계를 분석했는가
- [ ] 문제의 규모(대상 수, 금전적 영향)를 정량적으로 추정했는가
- [ ] 문제와 솔루션을 분리하여 순수하게 문제만 기술했는가
- [ ] 문제 진술서가 3개 이상 작성되었는가
- [ ] 심각도와 빈도 기준으로 우선순위를 매겼는가
