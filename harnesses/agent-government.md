# 공무원 에이전트 — 법적 검증 테스트

## 역할 정의

공무원 에이전트는 **사이트 사용자가 아닙니다.**
하네스 엔지니어링 플랫폼에서 산출된 구조검토서를 받았을 때,
**건축허가 심사 공무원 관점에서 법적으로 문제가 없는지** 테스트하는 역할입니다.

```
공무원은 플랫폼 밖에 있다

  [하네스 플랫폼]                    [공무원]
  고객 → AI 검토 → 기술사 날인       │
         ↓                           │
  구조검토서 출력물  ───── 제출 ────→ 심사
                                      │
                                결과: 적합/보완/반려
```

---

## 공무원이 검증하는 것 (20개 시나리오)

하네스 플랫폼이 **출력하는 검토서**와 **서비스 구조**가
건축법, 기술사법, 건설기술진흥법에 위반되지 않는지를 심사합니다.

### A. 기술사 날인 적법성 (G01~G05)

| ID | 검증 항목 | 기준 법령 | 판정 기준 |
|----|---------|----------|----------|
| G01 | 기술사가 실질적으로 검토했는가? | 국가기술자격법 제15조의2 | 명의대여가 아닌 실질 검토 여부 |
| G02 | 기술사사무소 개설등록이 되어있는가? | 기술사법 제6조 | 사무소 등록 없이 날인 시 위법 |
| G03 | 기술사 1인당 일 처리건수가 적정한가? | (실질검토 입증) | 1일 5건 이하 제한 준수 |
| G04 | 검토 소요시간이 합리적인가? | (형식적 검토 방지) | 건당 30분 이상 소요 |
| G05 | 기술사 자격 종목이 적합한가? | 기술사법 | 건축구조/토목구조 기술사 여부 |

### B. 구조검토서 법적 요건 (G06~G10)

| ID | 검증 항목 | 기준 법령 | 판정 기준 |
|----|---------|----------|----------|
| G06 | 구조안전확인서 양식이 적합한가? | 건축법 시행령 제32조 | 별지 제1호/2호/3호 서식 준수 |
| G07 | KDS/KCS 최신 기준을 적용했는가? | KDS 41 00 00 등 | 시행 중인 최신 기준 적용 여부 |
| G08 | 내진설계 대상이면 확인서가 있는가? | 건축법 제48조 | 3층 이상 내진설계 의무 |
| G09 | 기술사 서명날인이 있는가? | 건축법 시행령 제91조의3 | 구조도에 기술사 서명날인 의무 |
| G10 | 구조도와 계산서가 일치하는가? | (설계 정합성) | 도면과 계산서 수치 일치 |

| ID | 검증 항목 | 기준 법령 | 판정 기준 |
|----|---------|----------|----------|

| G12 | AI는 도구이고 기술사가 주체인가? | (판례 기준) | midas 등과 동일한 도구 구조 |
| G13 | 기술사가 매 건 개별 확인하는가? | (실질 검토 요건) | 일괄 날인이 아닌 건별 확인 |

| G20 | 검토서 위변조 방지 장치가 있는가? | (신뢰성) | 검증번호/QR/전자서명 존재 |

## 테스트 수행 (사이트 기반)

공무원은 사이트를 직접 사용하지 않지만,
**사이트에 표시된 정보를 근거로** 법적 적합성을 판단합니다.

### 검증 방법

```
G01~G05: /engineer/reviews/[id] 페이지에서 확인
  → 기술사 정보, 검토 소요시간, 체크리스트 확인

G06~G10: /dashboard/projects/[id] 페이지에서 확인
  → AI 검토 결과, 적용 기준, 기술사 날인 정보

G11~G15: 사이트 전체 구조에서 확인
  → 서비스 흐름, 기술사 역할, 이용약관

G16~G20: /engineer/reviews/[id] + /services 페이지에서 확인
  → 검토 항목 완전성, 안전율, 하중 산정
```
