# 아이디어 생성

발견한 문제를 기반으로 구체적인 솔루션 아이디어를 도출한다. 다양한 발상 기법을 활용하여 가능한 많은 후보를 만든 후, 정부사업 적합성까지 고려하여 유망한 아이디어를 선별한다.

---

## 규칙

1. **문제 기반 솔루션**: 반드시 앞 단계에서 정의한 문제를 해결하는 방향이어야 한다
2. **기술 적용 가능성**: 현재 기술로 구현 가능한 수준이어야 한다 (3-5년 내 실현)
3. **정부사업 적합성 고려**: 기술혁신성, 사업화 가능성, 사회적 가치를 동시에 충족해야 한다
4. **양 우선, 질 후순**: 먼저 최대한 많은 아이디어를 생성한 후 필터링한다
5. **차별점 필수**: 기존 솔루션 대비 명확한 차별화 포인트가 있어야 한다

---

## 아이디어 발상법

### 1. SCAMPER 기법
기존 솔루션이나 유사 제품을 기반으로 7가지 관점에서 변형한다.

| 관점 | 질문 | 예시 |
|------|------|------|
| **S**ubstitute (대체) | 무엇을 다른 것으로 바꿀 수 있는가? | 전문가 진단 → AI 자동 진단 |
| **C**ombine (결합) | 두 가지를 합치면? | IoT센서 + AI분석 = 예지보전 |
| **A**dapt (적용) | 다른 분야의 것을 가져오면? | 핀테크 신용평가 → 기술신용평가 |
| **M**odify (변형) | 크기/형태/방식을 바꾸면? | 대형장비 → 모바일/SaaS화 |
| **P**ut to other use (용도변경) | 다른 용도로 쓸 수 있는가? | 드론 촬영 → 드론 시설물 점검 |
| **E**liminate (제거) | 무엇을 없앨 수 있는가? | 중간 유통 제거 → D2C 플랫폼 |
| **R**earrange (재배열) | 순서/구조를 바꾸면? | 사후 관리 → 사전 예방 전환 |

### 2. 역발상
기존 상식이나 관행의 반대로 생각한다.

```
기존: 전문가가 직접 방문하여 컨설팅한다
역발상: 비전문가도 할 수 있도록 AI가 가이드한다

기존: 고객이 서비스를 찾아간다
역발상: 서비스가 고객에게 찾아간다 (자동 감지/추천)

기존: 대기업 솔루션을 축소하여 중소기업에 제공한다
역발상: 중소기업 전용으로 처음부터 설계한다
```

### 3. 기술-시장 매트릭스
보유/활용 가능한 기술과 타겟 시장을 교차하여 기회를 발견한다.

```
           | 제조업 | 유통/물류 | 의료/헬스 | 농업 | 교육 |
  ---------|--------|----------|----------|------|------|
  AI/ML    |        |          |          |      |      |
  IoT      |        |          |          |      |      |
  블록체인  |        |          |          |      |      |
  AR/VR    |        |          |          |      |      |
  로보틱스  |        |          |          |      |      |

→ 각 셀에 가능한 솔루션 아이디어를 채운다
→ 빈 칸(아직 솔루션이 없는 곳)이 블루오션 기회
```

### 4. 유사 서비스 벤치마킹
국내외 유사 서비스를 조사하여 영감을 얻는다.

- **해외 선행 사례**: 해외에는 있지만 국내에는 없는 서비스
- **타 산업 적용 사례**: 다른 산업에서 성공한 모델을 우리 산업에 적용
- **실패 사례 분석**: 왜 실패했는지 분석하여 개선점 도출
- **조사 채널**: Product Hunt, CB Insights, Crunchbase, TechCrunch

## 아이디어 카드 템플릿

각 아이디어를 아래 형식으로 작성한다:

```
### 아이디어 #N: [아이디어명]

**해결하는 문제**:
- 문제 진술서 #N 참조
- 핵심 페인포인트 한 줄 요약

**솔루션 개요**:
- 서비스/제품 형태: (SaaS / 하드웨어 / 앱 / 플랫폼 등)
- 작동 방식 2-3문장 설명
- 핵심 가치 제안 (Value Proposition)

**핵심 기술**:
- 주요 기술 스택 (예: 컴퓨터비전, NLP, IoT 등)
- 기술 성숙도: (이미 검증됨 / 연구단계 / 개발 필요)
- 기술 확보 방안: (자체개발 / 오픈소스 활용 / 외부 협력)

**타겟 고객**:
- 1차 타겟: (초기 시장)
- 확장 타겟: (성장 시장)
- 예상 고객 수: (TAM/SAM/SOM 간략 추정)

**수익모델 초안**:
- 과금 방식: (구독 / 건당 / 프리미엄 / 라이선스)
- 예상 객단가: (월 OO만원)
- BEP 도달 조건: (고객 N개사 확보 시)

**차별점**:
- 기존 대비 차별화 포인트 2-3가지
- 경쟁 우위 확보 방안 (기술 / 데이터 / 네트워크 등)
```

## 정부사업 적합성 사전 체크

아이디어가 정부지원사업에 적합한지 간단히 점검한다.

### 기술혁신성
- [ ] 기존 기술 대비 새로운 접근 방식이 있는가
- [ ] 특허 출원 또는 기술적 진입장벽이 가능한가
- [ ] 국가 전략기술 또는 신기술 분야에 해당하는가

### 사업화 가능성
- [ ] 명확한 수익모델이 존재하는가
- [ ] 타겟 시장 규모가 충분한가 (최소 100억 이상)
- [ ] 3년 내 매출 발생이 현실적인가

### 사회적 가치
- [ ] 일자리 창출 효과가 있는가
- [ ] 사회문제 해결에 기여하는가
- [ ] ESG 또는 SDGs 관련성이 있는가

---

## 산출물

1. **아이디어 카드 3-5개**: 위 템플릿에 따라 상세 작성
2. **솔루션 컨셉 요약표**: 아이디어명, 문제, 솔루션, 기술, 차별점을 한 페이지 표로 정리
3. **정부사업 적합성 사전 평가**: 각 아이디어별 기술혁신성/사업화/사회적 가치 간략 평가

---

## 체크리스트

- [ ] 모든 아이디어가 정의된 문제에서 출발했는가
- [ ] 최소 2가지 이상의 발상 기법을 활용했는가
- [ ] 각 아이디어에 명확한 차별점이 있는가
- [ ] 기술 실현 가능성을 검토했는가
- [ ] 수익모델 초안이 작성되었는가
- [ ] 타겟 고객이 구체적으로 정의되었는가
- [ ] 정부사업 적합성을 사전 점검했는가
- [ ] 아이디어 카드가 3개 이상 작성되었는가
- [ ] 유사 서비스/경쟁사를 조사했는가
