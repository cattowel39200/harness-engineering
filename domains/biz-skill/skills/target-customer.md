# 고객 정의

시장분석에서 도출한 고객 세그먼트를 구체적인 페르소나로 발전시키고, 고객의 니즈를 기능적/감정적/사회적 차원으로 매핑하여 제품 설계와 마케팅의 기반을 확보한다.

---

## 규칙

1. **B2B/B2G/B2C 구분**: 고객 유형에 따라 분석 프레임이 다르다. 반드시 먼저 구분한다
2. **구체적 페르소나**: "30대 직장인" 수준이 아닌, 이름/직업/목표/불만까지 구체화한다
3. **구매 의사결정 과정**: 누가, 언제, 어떤 기준으로 구매를 결정하는지 명확히 한다
4. **최소 2-3개 페르소나**: 하나의 페르소나에만 의존하지 않고 주요 고객군을 복수 정의한다
5. **지불의향 포함**: 각 페르소나별로 예상 지불 가능 금액과 근거를 반드시 포함한다

---

## 고객 세그먼트 분류

### 1단계: 대분류

| 분류 기준 | 구분 | 특징 |
|-----------|------|------|
| 거래 대상 | B2C (일반 소비자) | 개인 구매 결정, 감성 중시, 빠른 결정 |
| | B2B (기업) | 조직 구매 결정, ROI 중시, 긴 영업주기 |
| | B2G (정부/공공) | 조달/입찰, 규격 충족, 예산 주기 |

### 2단계: 세분화 기준

**B2C 세분화**

| 기준 | 항목 |
|------|------|
| 인구통계 | 연령, 성별, 소득, 거주지, 직업 |
| 행동 | 사용 빈도, 구매 채널, 기기, 앱 사용 패턴 |
| 심리 | 가치관, 라이프스타일, 기술 수용도 |
| 니즈 | 해결하려는 문제, 원하는 결과, 불편사항 |

**B2B 세분화**

| 기준 | 항목 |
|------|------|
| 기업 특성 | 산업, 규모(매출/인원), 지역, 업력 |
| 구매 특성 | 예산 규모, 구매 주기, 의사결정 구조 |
| 기술 환경 | 현재 시스템, 디지털 성숙도, IT 인력 |
| 니즈 | 비용 절감, 효율 향상, 규제 대응, 매출 증대 |

## 고객 니즈 매핑

각 페르소나별로 니즈를 3가지 차원으로 분류한다.

| 니즈 차원 | 설명 | 예시 |
|-----------|------|------|
| 기능적 니즈 | 실질적 문제 해결, 업무 효율화 | 처리 시간 50% 단축, 오류율 감소 |
| 감정적 니즈 | 심리적 만족, 불안 해소 | 안심감, 편리함, 스트레스 감소 |
| 사회적 니즈 | 대외 이미지, 관계, 규범 충족 | 혁신 기업 이미지, 규제 준수, 동료 인정 |

### 니즈맵 템플릿

```
페르소나: [이름]

기능적 니즈 (MUST HAVE)
  1.
  2.
  3.

감정적 니즈 (SHOULD HAVE)
  1.
  2.

사회적 니즈 (NICE TO HAVE)
  1.
  2.

→ 우리 솔루션이 해결하는 니즈: [상위 3개 매칭]
→ 미해결 니즈 (향후 로드맵): [1-2개]
```

---

## 산출물

1. **고객 세그먼트 정의서**: B2B/B2G/B2C 구분 + 세분화 기준
2. **페르소나 2-3개**: 각 주요 고객군의 구체적 페르소나
3. **니즈맵**: 페르소나별 기능적/감정적/사회적 니즈 매핑
4. **구매 의사결정 플로우**: 각 페르소나의 인지→검토→구매→사용 과정

---

## 체크리스트

- [ ] B2B/B2G/B2C 고객 유형을 명확히 구분했는가
- [ ] 페르소나를 최소 2-3개 작성했는가
- [ ] 각 페르소나에 이름, 직업, 목표, 불만이 구체적으로 기술되어 있는가
- [ ] 지불 의향(금액)과 그 근거가 포함되어 있는가
- [ ] 구매 의사결정 과정(누가, 언제, 어떤 기준으로)이 명확한가
- [ ] B2B의 경우 담당자 직급과 예산 확보 방법이 있는가
- [ ] B2G의 경우 조달 방식과 예산 편성 시기가 있는가
- [ ] 니즈를 기능적/감정적/사회적 3차원으로 분류했는가
- [ ] 우리 솔루션과 고객 니즈의 매칭이 명확한가
- [ ] 심사위원에게 "이 고객이 실제로 구매할 것"임을 설득할 수 있는가
