# 구조계산서 작성 수준 기준

이 스킬은 구조계산서를 **어느 수준까지 상세하게 작성해야 하는지** 기준을 정의한다.
특정 목차를 강제하는 것이 아니라, **어떤 구조물이든 동일한 깊이와 완성도**로 작성하도록 한다.

---

## 핵심 원칙

**기술사가 이 계산서만 보고 검증할 수 있어야 한다.**
기술사가 별도로 계산을 다시 하지 않아도, 계산서에 적힌 산식을 따라가면 결과를 확인할 수 있어야 한다.

---

## 작성 수준 기준

### 1. 설계조건은 빠짐없이 정리한다

```
해야 할 것:
  - 구조물 제원 (모든 치수를 표로)
  - 재료 강도 (fck, fy + 기준)
  - 토질/지반 조건 (γ, φ, c, qa, μ)
  - 강도감소계수 (φf, φv, φa + 기준 조항)
  - 피복두께 (부재별 + 기준 조항)
  - 하중 조건 (재하, 수위, 내진 등)
  - 단면 스케치 또는 제원도

하지 말 것:
  ✗ "일반적인 값 사용" — 구체적 수치 명시
  ✗ 설계조건 일부 생략
```

### 2. 모든 하중은 산출 과정을 보인다

```
해야 할 것:
  - 자중: 부분별 면적 × 단위중량 = 무게 (각각 산식)
  - 토압: 토압계수 산정 → 합력 산정 → 작용점 (6단계)
  - 수압: 수심 → 정수압 → 합력 → 작용점 (6단계)
  - 지진토압: M-O 공식 → KAE → PAE (6단계)
  - 하중조합: CASE별 ΣV, ΣH, ΣMR, ΣMO 표

하지 말 것:
  ✗ "자중 = 42.24 kN/m" (과정 없이 결과만)
  ✗ "토압 합력 Pa = 74.9 kN/m" (Ka 산정 과정 없이)
```

### 3. 안정 검토는 CASE별로 전부 쓴다

```
해야 할 것:
  - 검토할 CASE를 모두 정의 (상시, 홍수시, 지진시 등)
  - 각 CASE마다 전도/활동/지지력을 각각 6단계로
  - 전도 기준점이 CASE마다 다르면 각각 명시
  - CASE별 결과를 종합표로 정리

하지 말 것:
  ✗ "상시만 검토" (홍수시/지진시 해당되면 전부 검토)
  ✗ "마찬가지로 지진시도 OK" (산식 없이 결론만)
```

### 4. 부재 검토는 제원표부터 시작한다

```
해야 할 것:
  - 제원표 먼저 (두께, d, 배근, As, fck, fy)
  - 설계하중 산정 (하중계수 적용, LCB별)
  - 휨강도: Mu → a → Mn → φMn → 판정 (6단계 전부)
  - 전단강도: Vu → Vc → φVc → 판정 (6단계 전부)
  - 소요 철근량: 소요As → 최소As → 설계As → 배근안
  - 사용성: 균열모멘트, 사용응력, 소요간격

하지 말 것:
  ✗ "φMn = 278.02 kN·m" (a, Mn 산정 과정 없이)
  ✗ "전단 OK" (Vu, Vc 수치 없이)
  ✗ 제원표 없이 바로 산식 시작
```

### 5. 검토 대상 단면은 모두 검토한다

```
해야 할 것:
  - 벽체: 하단 + 중간 (단면이 변하면 각각)
  - 저판: 뒤꿈치 + 앞꿈치 (각각 캔틸레버)
  - RC건축물: 슬래브 + 보 + 기둥 + 기초 (각각)
  - 층별로 대표 부재가 다르면 각각

하지 말 것:
  ✗ "벽체 하단만 검토" (중간부 생략)
  ✗ "저판은 벽체와 유사" (산식 없이 생략)
```

### 6. 배근 상세는 시공 가능한 수준으로 쓴다

```
해야 할 것:
  - 최소/최대 철근비 확인
  - 정착길이 산정 (보정계수 포함)
  - 이음길이 산정 (Class A/B)
  - 배근 간격 확인 (최대간격 기준)
  - 배수공, 신축이음 등 부대시설 (해당 시)

하지 말 것:
  ✗ "배근 적정" (수치 없이)
  ✗ 정착/이음 생략
```

### 7. 종합판정은 전 항목을 표로 정리한다

```
해야 할 것:
  - 모든 검토항목을 한 표에: 항목/계산값/기준값/비율/판정
  - 안정검토(전도/활동/지지력) + 단면검토(휨/전단) + 배근 전부
  - FAIL 항목이 있으면 보완 방안 구체적 기재
  - 최종 결론 한 줄

하지 말 것:
  ✗ "전 항목 OK" (표 없이)
  ✗ FAIL 항목 보완안 미기재
```

### 8. 기술사 검토란은 반드시 포함한다

```
해야 할 것:
  - 기술사 정보란 (성명/자격/사무소)
  - 검토 의견란 (빈칸)
  - 확인 선서문
  - 무료 계산서: "기술사 미포함" 명시
  - 유료 날인: QR 검증번호 포함

하지 말 것:
  ✗ 기술사란 생략
```

## 금지 표현 목록

계산서에서 다음 표현이 나오면 작성 수준 미달이다:

| 금지 표현 | 이유 | 대신 할 것 |
|----------|------|----------|
| "계산하면" | 과정 생략 | 대입+과정을 전부 쓴다 |
| "마찬가지로" | 반복 산식 생략 | 매번 산식을 쓴다 |
| "위와 동일" | 참조 생략 | 해당 산식을 다시 쓴다 |
| "약", "대략", "정도" | 모호한 수치 | 정확한 값 + 단위 |
| "OK", "적정" | 근거 없는 판정 | 계산값 ≤ 기준값 → PASS |
| "검토 생략" | 항목 누락 | "해당 없음 (사유: ___)" |
| "참고" | 의무 미이행 | 기준 조항 직접 인용 |

## 체크리스트 (계산서 완성 후 자체 검증)

- [ ] 설계조건이 빠짐없이 정리되었는가 (재료/토질/지반/하중/내진)
- [ ] 모든 하중에 산출 과정(6단계)이 있는가
- [ ] 검토 CASE를 전부 정의하고 각각 검토했는가
- [ ] 모든 부재 검토에 제원표가 먼저 있는가
- [ ] 휨/전단 검토에 6단계 산식이 전부 있는가
- [ ] 검토 대상 단면을 모두 검토했는가 (하단+중간, 뒤꿈치+앞꿈치)
- [ ] 배근 상세(철근비/정착/이음)가 포함되었는가
- [ ] 종합판정표에 전 항목이 있는가
- [ ] 기술사 검토란이 포함되었는가
- [ ] 금지 표현이 하나도 없는가
- [ ] 모든 수치에 단위가 있는가
- [ ] 모든 공식에 KDS 조항번호가 있는가
