# 하네스 엔지니어링 — 테스트 에이전트 시나리오

3역할(관리자, 고객, 기술사)이 서로 연결되어 전체 서비스 흐름을 완성합니다.
각 에이전트는 자기 역할의 목표를 수행하고, 다른 역할의 액션을 트리거합니다.

---

## 에이전트 1: 관리자 (Admin Agent)

### 역할
플랫폼 전체를 관리하고, 기술사를 승인하고, 품질과 정산을 감독합니다.

### 테스트 시나리오

#### 시나리오 A: 기술사 승인 프로세스
```
목표: 신규 기술사 "최기술"의 가입 승인

수행 순서:
1. /admin/dashboard 접속
   → "기술사 승인 대기 1건" 알림 확인

2. /admin/engineers 이동
   → "최기술" (토목구조기술사, 2018-5678) 상태: "승인 대기" 확인

3. 자격 검증 체크리스트:
   □ 기술사 자격증 확인 (토목구조기술사 2018-5678)
   □ 기술사사무소 개설등록증 확인 (최기술 엔지니어링)
   □ 경력 확인 (8년, 토목설계사무소)
   □ 전문분야 확인 (옹벽, 교량, 가설구조물)

4. 판단:
   - 자격요건 충족 → [승인] 클릭
   - 서류 미비 시 → [거절] + 사유 입력

5. 승인 후 확인:
   → "최기술" 상태가 "승인"으로 변경
   → 기술사가 검토 대기열에서 건을 수락할 수 있게 됨

예상 결과:
✅ 기술사 승인 완료
✅ 기술사 네트워크에 1명 추가
```

#### 시나리오 B: 프로젝트 모니터링
```
목표: 전체 프로젝트 현황 파악 및 이상 감지

수행 순서:
1. /admin/dashboard 접속
   → 통계 확인: 총 회원 5명, 기술사 2명, 이번달 매출 198만원

2. /admin/projects 이동
   → 전체 프로젝트 4건 확인
   → 상태별 필터: AI검토중 1건, 기술사검증중 1건, 완료 1건, 결제대기 1건

3. 이상 감지 체크:
   □ 48시간 이상 "기술사 검증 중"인 건이 있는가?
   □ 결제 대기 상태로 24시간 이상 방치된 건이 있는가?
   □ 기술사 1인당 일 5건 초과하지 않았는가?

4. 조치:
   - 지연 건 발견 시 → 기술사에게 알림 / 재배정
   - 미결제 건 → 고객에게 안내

예상 결과:
✅ 전체 현황 파악 완료
✅ 이상 건 0건 확인
```

#### 시나리오 C: 정산 처리
```
목표: 완료된 프로젝트의 기술사 정산 집행

수행 순서:
1. /admin/payments 접속 → "기술사 정산" 탭

2. 정산 대기 확인:
   → p2 "△△ 물류센터 기초 설계검토" - 박구조 기술사 - 125,000원 - 대기 중

3. 정산 조건 확인:
   □ 고객이 "완료 확인" 했는가?
   □ 기술사가 실질 검증을 수행했는가? (검토시간 30분 이상)
   □ 환불/분쟁 요청이 없는가?

4. 정산 집행:
   → [정산 처리] 클릭 → 기술사 계좌로 이체

예상 결과:
✅ 기술사 정산 완료
✅ 정산 상태: "대기" → "완료"
```

## 에이전트 2: 고객 (Client Agent)

### 역할
구조검토를 의뢰하고, 결과를 확인하고, 비용을 결제합니다.

### 테스트 시나리오

#### 시나리오 A: 신규 구조검토 의뢰 (풀 프로세스)
```
목표: RC 구조검토를 의뢰하고 완료까지 진행

수행 순서:
1. /register 접속 → "의뢰인" 선택하여 회원가입
   이름: 김건설, 이메일: client@test.com

2. /dashboard 접속 → 대시보드 확인
   → "총 프로젝트 0건" 초기 상태

3. /dashboard/projects/new 이동 → 새 의뢰 시작

4. Step 1 - 구조물 유형 선택:
   → "RC 구조검토" 선택 (기본 50만원~)

5. Step 2 - 프로젝트 정보:
   제목: "○○ 아파트 신축 RC 구조검토"
   설명: "지상 15층 아파트 RC 구조검토. 슬래브, 보, 기둥, 기초 전체."

6. Step 3 - 파일 업로드:
   → 구조평면도.dwg (2.4MB)
   → 설계조건.xlsx (180KB)

7. Step 4 - 견적 확인 + 결제:
   기본료: 800,000원
   부가세: 80,000원
   합  계: 880,000원
   → [결제하기] 클릭

8. 결제 완료 → 프로젝트 상태: "AI 검토 중"
   → /dashboard/projects/[id] 에서 진행상태 확인

--- (AI 검토 + 기술사 검증 후) ---

9. 알림: "검토서가 준비되었습니다"
   → /dashboard/projects/[id] 이동

10. 검토서 확인:
    □ AI 검토 결과 확인 (KDS 기준 검증 내역)
    □ 기술사 날인 검토서 다운로드
    □ 검토 의견 확인

11. 판단:
    - 만족 → [완료 확인] 클릭
    - 수정 필요 → [수정 요청] + 사유 입력

12. 완료 확인 후:
    → [리뷰 작성] → 별점 5점 + "당일 납품에 품질도 훌륭합니다"

예상 결과:
✅ 구조검토 의뢰 → 결제 → AI 검토 → 기술사 검증 → 납품 → 리뷰 완료
✅ 결제 내역: /dashboard/payments 에 880,000원 기록
```

#### 시나리오 B: 수정 요청
```
목표: 검토 결과에 대해 수정 요청 후 재납품 받기

수행 순서:
1. /dashboard/projects/[p2] 접속
   → 상태: "기술사 검증 중"

2. (기술사 검증 완료 후) 검토서 확인
   → 기초 두께가 설계 의도와 다르게 검토된 부분 발견

3. [수정 요청] 클릭
   → 사유: "매트기초 두께 600mm → 800mm로 변경된 설계를 반영 요청"

4. 상태 변경: "기술사 검증 중" → "수정 요청"
   → 기술사에게 수정 사유 전달

5. (기술사 수정 후) 재납품 확인
   → [완료 확인]

예상 결과:
✅ 수정 요청 → 기술사 재검토 → 재납품 → 최종 완료
```

#### 시나리오 C: 결제 내역 확인
```
목표: 전체 결제 내역과 비용 확인

수행 순서:
1. /dashboard/payments 접속
2. 확인:
   → 총 결제: 3건, 1,980,000원
   → 완료: 3건
   → 환불: 0건
3. 개별 건 클릭 → 프로젝트 상세 이동

예상 결과:
✅ 전체 결제 이력 확인 완료
```

## 에이전트 3: 기술사 (Engineer Agent)

### 역할
AI 검토 결과를 전문가로서 검증하고, 승인/수정/반려를 판단하여 날인합니다.

### 테스트 시나리오

#### 시나리오 A: 가입 및 자격 인증
```
목표: 기술사로 가입하고 자격 승인을 받기

수행 순서:
1. /register 접속 → "기술사" 선택

2. 기본 정보 입력:
   이름: 박구조
   이메일: engineer@test.com
   연락처: 010-9876-5432

3. 기술사 추가 정보:
   자격 종목: 건축구조기술사
   자격 번호: 2015-1234
   사무소명: 박구조 기술사사무소
   경력: 12년 (대형 구조설계사무소 10년 + 기술사사무소 2년)

4. 회원가입 완료 → 상태: "승인 대기"
   → "관리자 승인 후 서비스 이용이 가능합니다" 안내

--- (관리자 승인 후) ---

5. /engineer/dashboard 접속
   → "승인 완료! 검토 대기열을 확인하세요" 알림

예상 결과:
✅ 기술사 가입 → 관리자 승인 → 활동 시작
```

#### 시나리오 B: 구조검토 검증 수행 (핵심 시나리오)
```
목표: AI 검토 결과를 확인하고 전문가로서 승인/날인

수행 순서:
1. /engineer/dashboard 접속
   → "검토 대기 2건" 확인

2. /engineer/reviews 이동
   → 대기 목록:
     · p2 "△△ 물류센터 기초 설계검토" (기초, 125,000원)
     · p3 "□□ 옹벽 구조검토" (옹벽, 125,000원)

3. p2 "△△ 물류센터" [수락] 클릭
   → 상태: "검증 진행 중"

4. /engineer/reviews/p2 이동 (검토 수행 화면)

5. AI 검토 결과 확인 — 체크리스트 수행:

   ┌─────────────────────────────────────────────┐
   │  AI 구조검토 결과 — 기초 설계검토             │
   │  적용 기준: KDS 41 12 00 / KDS 11 50 00      │
   │                                               │
   │  ☑ 지반 지지력 검토 ─────── PASS (안전율 3.2) │
   │  ☑ 기초판 두께 적정성 ───── PASS (800mm)      │
   │  ☑ 배근 간격 기준 충족 ─── PASS (D22@200)     │
   │  ☑ 전도 안전율 ──────────── PASS (2.1 > 1.5)  │
   │  ☑ 활동 안전율 ──────────── PASS (1.8 > 1.5)  │
   │  ☑ 지지력 안전율 ────────── PASS (3.2 > 3.0)  │
   │  ☑ 침하량 검토 ──────────── PASS (12mm < 25mm) │
   │  ☐ 말뚝 수평지지력 ──────── N/A (직접기초)     │
   └─────────────────────────────────────────────┘

6. 기술사 전문 판단:
   □ AI 계산 결과가 설계조건과 일치하는가?
   □ 적용 기준 조항이 정확한가?
   □ 안전율이 기준 이상인가?
   □ 추가 검토가 필요한 항목은 없는가?

7. 검토 의견 작성:
   "AI 검토 결과 확인함. 지반지지력, 전도/활동 안전율 모두 기준 충족.
    기초판 두께 800mm, 배근 D22@200 적정.
    특이사항 없음. 승인합니다."

8. 판단:
   ─ 모두 적정 → [승인 (날인)] 클릭 (녹색)
   ─ 일부 수정 필요 → [수정 요청] + 사유 (노란색)
   ─ 심각한 오류 → [반려] + 사유 (빨간색)

9. [승인 (날인)] 클릭
   → "기술사 날인이 완료되었습니다"
   → 검토서에 기술사 서명·날인 추가
   → 프로젝트 상태: "기술사 검증 중" → "완료"
   → 고객에게 납품 알림

10. 검토 소요시간: 45분
    → 시간당 수입: 166,667원 (125,000 / 0.75시간)

예상 결과:
✅ AI 결과 확인 → 체크리스트 수행 → 전문 판단 → 승인·날인 완료
✅ 검토 이력 기록 (확인시간, 체크리스트, 의견, 판단)
```

#### 시나리오 C: 수정 요청 처리
```
목표: 고객의 수정 요청을 처리하여 재검토

수행 순서:
1. /engineer/reviews 접속
   → p2 상태: "수정 요청" 표시
   → 수정 사유: "매트기초 두께 600→800mm 변경 반영 요청"

2. /engineer/reviews/p2 이동
   → 수정된 설계조건 확인
   → AI가 재검토한 결과 확인

3. 재검증 수행:
   □ 변경된 두께(800mm)에 대한 검토 결과 확인
   □ 안전율 재확인
   □ 기타 영향 검토

4. 검토 의견: "변경 설계(t=800mm) 반영 확인. 안전율 충족. 승인."
   → [승인 (날인)]

예상 결과:
✅ 수정 요청 처리 → 재검증 → 재승인·날인 완료
```

#### 시나리오 D: 정산 확인
```
목표: 이번 달 수입과 정산 상태 확인

수행 순서:
1. /engineer/earnings 접속

2. 확인:
   총 수입: 200,000원
   정산 완료: 200,000원 (p1)
   정산 대기: 125,000원 (p2 - 완료 후 정산 예정)

3. [출금 신청] → 정산 완료 금액 출금 요청

예상 결과:
✅ 정산 내역 확인
✅ 출금 신청 완료
```

## 3역할 연동 테스트 시나리오 (E2E)

```
시간순서로 3역할이 어떻게 만나는지:

T1  [기술사] 회원가입 → 자격정보 제출 → 상태: 승인 대기
T2  [관리자] 기술사 자격 확인 → [승인] → 기술사 활동 가능
T3  [고객]   회원가입 → 로그인
T4  [고객]   새 의뢰 등록 (RC 구조검토, 파일 업로드)
T5  [고객]   견적 확인 → 결제 (880,000원 에스크로)
T6  [시스템] AI 구조검토 자동 수행 (하네스 엔진)
T7  [시스템] AI 검토 완료 → 기술사 배정 → 알림
T8  [기술사] 검토 대기열에서 건 [수락]
```
